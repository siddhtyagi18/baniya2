# Plan 05-05: Dashboard Sub-Components

## Objective

Build all dashboard sub-components used by BaniyaDashboard.vue: CostCard, SavingsCard, RoutingPie (Chart.js donut), AuditTable, and ProviderStatus.

## Requirements Addressed

- REQ-18: Dashboard sub-components
- REQ-25: Accurate savings display
- REQ-26: Provider status live updates

## Spec Reference

See `context.md` §13 (Dashboard UI) for exact layout and data requirements.

## Files to Create

- `apps/editor/src/components/dashboard/CostCard.vue`
- `apps/editor/src/components/dashboard/SavingsCard.vue`
- `apps/editor/src/components/dashboard/RoutingPie.vue`
- `apps/editor/src/components/dashboard/AuditTable.vue`
- `apps/editor/src/components/dashboard/ProviderStatus.vue`

## Tasks

### Task 1: CostCard.vue
- Summary card showing: Total spend (₹ X.XX / $Y.YY), If-all-cloud cost, Executions count
- Props: `totalCostUSD`, `totalCostINR`, `hypotheticalCost`, `executionCount`
- Always show both ₹ and $ — never one without the other

### Task 2: SavingsCard.vue
- Savings card: Baniya saved ₹ X.XX (Z%)
- Green accent when savings > 0
- Props: `savingsUSD`, `savingsINR`, `percent`

### Task 3: RoutingPie.vue
- Chart.js donut chart: % local / hybrid / cloud
- Colours: local=#10B981, hybrid=#F59E0B, cloud=#3B82F6
- Legend below chart
- Props: `byRoute: Record<RoutingTarget, number>`

### Task 4: AuditTable.vue
- Columns: Time | Workflow | Node | Sensitivity | Route | Model | Cost | Latency
- Sensitivity badge: critical=red, private=amber, internal=blue, public=green
- Route pill: local=green, hybrid=amber, cloud=blue
- Sortable columns, filter bar (sensitivity dropdown, workflow dropdown)
- Pagination: 20 rows/page
- Props: `rows: AuditRow[]`, `total: number`, events for sort/filter/page

### Task 5: ProviderStatus.vue
- Horizontal bar: Ollama ● status, LM Studio ● status, OpenAI ● status, etc.
- Green dot = running/configured, red dot = offline/missing key
- Auto-polls `/api/baniya/providers/status` every 10 seconds
- Uses providers store

### Task 6: Wire into BaniyaDashboard.vue
- Update BaniyaDashboard to use these components instead of inline rendering
- Connect to API via `baniya.ts` API layer
- Add chart.js import

## Verification

- Dashboard loads with 4 summary cards
- Donut chart renders with routing breakdown
- Audit table shows data with correct badge colours
- Provider status updates every 10 seconds
