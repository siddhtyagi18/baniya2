# Plan 05-04: Missing Editor Components

## Objective

Build all missing Vue components for the editor canvas: NodePicker (left sidebar), NodeConfigPanel (right drawer), and shared UI components (Topbar, Modal, Badge, Spinner, EmptyState, EdgeLabel, MiniExecutionBadge).

## Requirements Addressed

- REQ-15: NodePicker sidebar
- REQ-16: NodeConfigPanel
- REQ-17: Shared UI components

## Spec Reference

See `context.md` §11 (Node Registry), §12 (Canvas UI), §14 (Design Tokens).

## Files to Create

### Canvas Components
- `apps/editor/src/components/canvas/NodePicker.vue` — Left sidebar (280px), search box, nodes grouped by category (Triggers/AI/Logic/Data/Output), drag-to-add
- `apps/editor/src/components/canvas/NodeConfigPanel.vue` — Right panel (320px, slides in), renders config fields by type (text/textarea/number/boolean/select/code/expression)
- `apps/editor/src/components/canvas/EdgeLabel.vue` — Label on edges showing handle name
- `apps/editor/src/components/canvas/MiniExecutionBadge.vue` — Cost badge on node during/after execution

### Shared Components
- `apps/editor/src/components/shared/Topbar.vue` — Top navigation bar with workflow name, active toggle, save/run buttons
- `apps/editor/src/components/shared/Modal.vue` — Generic modal with overlay, close button, slot content
- `apps/editor/src/components/shared/Badge.vue` — Coloured badge (sensitivity levels, route types)
- `apps/editor/src/components/shared/Spinner.vue` — Loading spinner animation
- `apps/editor/src/components/shared/EmptyState.vue` — Empty state illustration + CTA button

## Tasks

### Task 1: NodePicker.vue
- Import node registry from `@baniya/nodes`
- Group nodes by category with colour-coded left border
- Search box filters by name
- Each node card shows icon (SVG path), name, description
- Drag support: `@dragstart` sets node type data for vue-flow `onDrop`

### Task 2: NodeConfigPanel.vue
- Reads selected node's type → looks up configSchema from registry
- Renders field by type:
  - `text` → `<input type="text">`
  - `textarea` → `<textarea>` with auto-resize
  - `number` → `<input type="number">` with min/max
  - `boolean` → toggle switch
  - `select` → `<select>` with options
  - `code` → Monaco editor from CDN
  - `expression` → single-line input with `{{ }}` hint
- Two-way binding: changes update workflow store immediately
- Close button top-right

### Task 3: Shared components
- Topbar: workflow name (editable inline), active toggle, Save btn, Run btn, execution status
- Modal: `<teleport to="body">`, overlay click closes, ESC closes
- Badge: props `text`, `color`, `variant` (solid/outline)
- Spinner: CSS-only animation, configurable size
- EmptyState: illustration slot, title, description, action button

### Task 4: EdgeLabel + MiniExecutionBadge
- EdgeLabel: positioned along edge, shows handle name
- MiniExecutionBadge: shows "$0.000" or "free" after execution, status icon

### Task 5: Integrate into WorkflowEditor
- Add NodePicker to left panel
- Add NodeConfigPanel to right panel (show on node click)
- Add Topbar to canvas toolbar area
- Wire execution badges to WebSocket data

## Verification

- Open WorkflowEditor → NodePicker visible on left with all 20 node types
- Click a node → NodeConfigPanel slides in from right with correct fields
- Drag a node from picker onto canvas → node appears
- All components use design tokens from `index.css`
