# Plan 04-04: Zod Validation on All Routes

## Objective

Add Zod input validation schemas to every API route in `apps/server/src/routes/`. This is a security requirement — all user input must be validated before processing.

## Requirements Addressed

- REQ-14: Zod validation on ALL API routes

## Files to Modify

- `apps/server/src/routes/auth.ts`
- `apps/server/src/routes/workflows.ts`
- `apps/server/src/routes/executions.ts`
- `apps/server/src/routes/baniya.ts`
- `apps/server/src/routes/webhooks.ts`
- `apps/server/package.json` (add zod dependency if not present)

## Tasks

### Task 1: Add Zod dependency
Check if `zod` is in server's `package.json`. If not, add it.

### Task 2: Create validation schemas
Create `apps/server/src/validation/schemas.ts`:
- `loginSchema`: `{ email: z.string().email(), password: z.string().min(6) }`
- `registerSchema`: `{ email: z.string().email(), password: z.string().min(6), name: z.string().min(1) }`
- `createWorkflowSchema`: `{ name: z.string(), description: z.string().optional(), definition: z.record(z.unknown()).optional() }`
- `updateWorkflowSchema`: `{ name: z.string(), description: z.string().optional(), definition: z.record(z.unknown()), active: z.boolean().optional() }`
- `toggleActiveSchema`: `{ active: z.boolean() }`
- `classifySchema`: `{ payload: z.unknown() }`
- `routeSchema`: `{ payload: z.unknown(), prompt: z.string(), config: z.record(z.unknown()).optional() }`
- `paginationSchema`: `{ page: z.coerce.number().optional(), limit: z.coerce.number().optional() }`

### Task 3: Create validation middleware
Create `apps/server/src/middleware/validate.ts`:
```typescript
export function validate(schema: ZodSchema) {
  return (req, res, next) => {
    const result = schema.safeParse(req.body);
    if (!result.success) {
      return res.status(400).json({ error: 'Validation error', details: result.error.issues });
    }
    req.body = result.data;
    next();
  };
}
```

### Task 4: Apply validation to all routes
Add `validate(schema)` middleware to each route handler.

### Task 5: Add query parameter validation
For GET routes with query params (cost-summary, audit, executions), validate query parameters with `z.coerce`.
