# Plan 06-04: Demo Workflow + End-to-End Verification

## Objective

Verify the demo workflow seed runs end-to-end. Test all routing paths (local, hybrid, cloud). Verify audit log populates and dashboard shows accurate cost savings.

## Requirements Addressed

- REQ-20: Demo workflow runs end-to-end
- REQ-21: Aadhaar payload routes to local, never cloud
- REQ-22: Clean business payload routes to cloud
- REQ-23: Mixed payload takes hybrid path, PII scrubbed
- REQ-24: Audit log shows one row per node execution
- REQ-25: Dashboard shows accurate savings vs all-cloud
- REQ-28: WebSocket reconnects on disconnect

## Spec Reference

See `context.md` §19 (Demo Workflow) for the exact workflow graph.

## Tasks

### Task 1: Verify Demo Workflow Seed
- Check `apps/server/src/seed.ts` creates the demo workflow on first startup
- Verify workflow structure matches spec:
  ```
  trigger.manual → ai.classify → [private: ai.llm local] / [public: ai.llm cloud] → logic.merge → data.set → output.response
  ```

### Task 2: Test — Aadhaar Payload → Local
- POST to `/api/workflows/:id/execute` with payload containing Aadhaar number
- Verify: classifier returns `critical`, router blocks cloud, routes to local
- Verify: audit log entry shows `routing_decision: local`, `sensitivity_level: critical`

### Task 3: Test — Clean Business Text → Cloud
- POST with payload: `{ "text": "Quarterly revenue increased by 15% compared to last year" }`
- Verify: classifier returns `public`, routes to cloud
- Verify: audit log shows `routing_decision: cloud`, `sensitivity_level: public`

### Task 4: Test — Mixed Payload → Hybrid
- POST with payload containing business text + email address
- Verify: classifier returns `private`/`internal`, sanitizer replaces email with placeholder
- Verify: cloud receives sanitized text, response has originals re-injected

### Task 5: Test — Audit Log Completeness
- After executions, verify `/api/baniya/audit` returns one row per node
- Verify cost, latency, model fields are populated
- Verify no PII appears in audit log entries

### Task 6: Test — Dashboard Accuracy
- Call `/api/baniya/cost-summary`
- Verify `savingsVsAllCloudUSD` is calculated correctly
- Verify both ₹ and $ amounts are present

### Task 7: Test — WebSocket Reconnect
- Connect WebSocket, verify events stream during execution
- Disconnect, verify reconnect with exponential backoff
- Verify execution status updates resume after reconnect

## Verification

- [ ] Demo workflow runs complete — all nodes execute
- [ ] Aadhaar data never reaches cloud
- [ ] Clean text routes to cheapest cloud model
- [ ] Audit log is complete and PII-free
- [ ] Dashboard savings calculation matches manual calculation
- [ ] WebSocket reconnects automatically
