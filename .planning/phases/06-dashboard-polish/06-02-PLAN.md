# Plan 06-02: ExecutionDetail + Settings Polish

## Objective

Polish ExecutionDetail view with full node result timeline. Implement dark mode toggle in Settings view with localStorage persistence.

## Requirements Addressed

- REQ-19: Dark mode toggle in Settings
- REQ-27: Dark mode works across all views

## Spec Reference

See `context.md` §14 (Design Tokens) for dark mode CSS variables.

## Files to Modify

- `apps/editor/src/views/ExecutionDetail.vue`
- `apps/editor/src/views/Settings.vue`

## Tasks

### Task 1: ExecutionDetail — Node Result Timeline
- Show each node execution result in chronological order
- For each node: name, status (icon + colour), duration, cost
- If AI node: show model used, routing decision, sensitivity level
- Error nodes: show error message with expandable stack trace
- Overall execution summary at top: total cost, total time, status

### Task 2: Settings — Dark Mode Toggle  
- Toggle switch to enable/disable dark mode
- On toggle: add/remove `.dark` class on `<html>` element
- Persist to `localStorage.getItem('baniya-dark-mode')`
- Read on app startup (in `main.ts` or `App.vue`)

### Task 3: Settings — Provider Status Display
- Show current provider configuration status
- API key status: configured / missing for OpenAI, Anthropic, Gemini
- Local provider status: Ollama running/offline, LM Studio running/offline

### Task 4: Dark Mode for All Views
- Verify all views render correctly in dark mode
- Fix any hardcoded colours (replace with CSS variables)
- Test: Login, Register, WorkflowList, WorkflowEditor, Dashboard, Settings

## Verification

- Toggle dark mode in Settings → all views switch correctly
- Refresh page → dark mode persists
- ExecutionDetail shows complete node timeline with cost/routing info
