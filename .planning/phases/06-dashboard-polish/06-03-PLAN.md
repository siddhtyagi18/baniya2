# Plan 06-03: Auth Flow Polish

## Objective

Polish Login and Register views with proper error handling, loading states, and redirect logic. Ensure all API routes reject unauthenticated requests.

## Requirements Addressed

- REQ-29: All API routes reject unauthenticated requests with 401

## Files to Modify

- `apps/editor/src/views/Login.vue`
- `apps/editor/src/views/Register.vue`
- `apps/editor/src/stores/auth.ts`
- `apps/editor/src/api/client.ts`
- `apps/editor/src/router/index.ts`

## Tasks

### Task 1: Auth Store Enhancement
- Token management (localStorage read/write/clear)
- Auto-logout on 401 response (axios interceptor)
- User profile fetch on app load if token exists
- Login/register actions with error handling

### Task 2: Login/Register Polish
- Loading spinner during API call
- Error message display (invalid credentials, email taken, etc.)
- Redirect to `/workflows` after successful auth
- "Already have an account?" / "Create an account" links

### Task 3: Router Guards
- Navigation guard: if route has `meta.auth` and no token → redirect to `/login`
- If already logged in and visiting `/login` or `/register` → redirect to `/workflows`

### Task 4: Axios Interceptor
- Attach `Authorization: Bearer <token>` to all API requests
- On 401 response: clear token, redirect to `/login`

## Verification

- Visit `/workflows` without login → redirected to `/login`
- Login with valid credentials → redirected to `/workflows`
- Invalid credentials → error message shown
- API calls include auth token
